# Example Configuration Files

This directory contains example configuration and output files for the iCloud Drive Downloader.

## 🎯 Quick Start: Use the Wizard!

The easiest way to create a config is with the interactive wizard:

```bash
python3 icloud_downloader.py --wizard
```

Or use one of the built-in presets:

```bash
# Photos only
python3 icloud_downloader.py --preset photos

# Documents only  
python3 icloud_downloader.py --preset documents

# Quick test (first 50 files, 2 levels)
python3 icloud_downloader.py --preset quick-test

# Large files only (>100MB)
python3 icloud_downloader.py --preset large-files
```

### [large-files-config.json](large-files-config.json)
Configuration for large file downloads:

- Extended timeout (5 minutes)
- Higher retry count (10 attempts)
- Debug logging enabled

**Usage:**
```bash
python3 icloud_downloader.py --config examples/large-files-config.json
```

## Creating Your Own Config

1. Start with one of the examples above
2. Modify paths, patterns, and settings for your needs
3. Test with `--dry-run` first:
   ```bash
   python3 icloud_downloader.py --config your-config.json --dry-run
   ```
4. Run the actual download:
   ```bash
   python3 icloud_downloader.py --config your-config.json
   ```

## Saving Current Options

You can save your current command-line options to a config file:

```bash
python3 icloud_downloader.py \
  --workers 5 \
  --include "*.pdf" \
  --exclude "*/temp/*" \
  --max-depth 3 \
  --save-config my-config.json
```

This creates a reusable configuration file without starting a download.

## Tips

- Use `--dry-run` to preview what will be downloaded
- Start with conservative settings (fewer workers, lower depth)
- Use `--max-items` to test with a limited number of files first
- Review the generated manifest to understand what's been downloaded
- Check the JSONL log for detailed event tracking
